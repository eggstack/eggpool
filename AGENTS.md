## Pre-commit Checks (run before every commit)

```bash
uv run ruff format --check src/ tests/ scripts/
uv run ruff check src/ tests/ scripts/
uv run pyright src/ scripts/
uv run pytest
```

All four must pass with zero errors.

## Code Style

- Python 3.11+ with `from __future__ import annotations` in ALL files
- Type hints on all function signatures and return types
- Ruff: E, F, W, I, N, UP, B, A, SIM, TCH rules
- Pyright strict mode ‚Äî covers `src/` AND `scripts/` (not tests)
- Line length: 88 chars
- Use `NoReturn` for functions that never return (e.g., `sys.exit`)

- **Multi-provider architecture**: provider-suffixed model IDs (`model-id/provider-id`), `ProviderClientPool`, `OutboundClientManager`. See `architecture/README.md` ¬ß Multi-Provider Architecture.
- **Provider contracts**: `compose_provider_url()` is the single source of truth for upstream URLs. See `architecture/README.md` ¬ß Provider Contracts and ¬ß Provider Contract Rendering.
- **Protocol transcoding**: transparent request/response format conversion between OpenAI and Anthropic protocols. Phase 2 body translation, Phase 3 streaming SSE translation, Phase 4 routing eligibility widening, Phase 5 operator controls and docs, and Phase 6.1 tool-use/tool_calls body and streaming translation (including `pause_turn` sentinel handling and `stream_options.include_usage` lifting) are implemented in `src/eggpool/transcoder/` and `src/eggpool/request/coordinator.py`. - **Error hierarchy**: `AggregatorError` ‚Üí `UpstreamError` ‚Üí specific subclasses. `CapabilityError` for thinking/reasoning capability mismatches. See `architecture/README.md` ¬ß Error Hierarchy.

Request traces store `thinking_trace_json` on the `requests` table (migration `0039`). Exposed via `GET /api/stats/thinking` and included in `/api/stats/runtime` under `thinking_metrics`. See `plans/thinking_reasoning_phase_11_test_matrix.md`.
- **Catalog refresh semantics**: `CatalogService._fetch_and_process_account` returns an `AccountCatalogOutcome` enum (`SUCCESS_AUTHORITATIVE`, `SUCCESS_PARTIAL`, `SUCCESS_EMPTY`, `FAILED`, `SKIPPED`). Per-account cache updates run through `ModelCatalogCache.update_from_account(authoritative, allow_withdrawals)`, which is **non-destructive by default** ‚Äî both flags default to `False`. The destructive `mark_account_models_unavailable` step is gated on `authoritative=True AND allow_withdrawals=True`, so the cache layer itself enforces the invariant. `ModelsConfig.catalog_withdrawal_policy` (`preserve_until_health` (default), `confirmed_once`, `confirmed_twice`) governs when the service flips `allow_withdrawals=True`. `SUCCESS_PARTIAL` always forces withdrawal off because a partial response is never a complete withdrawal confirmation. The per-cycle INFO summary at `_log_refresh_summary` reports per-outcome counts so operators can spot catalog uncertainty without enabling debug logging. See `architecture/README.md` ¬ß Catalog Refresh Semantics.
- **Agent integrations**: `eggpool configsetup` generates configuration snippets for 11 coding agents (OpenCode, Claude Code, Aider, Codex, Qwen Code, Kilo, Continue, Cline, Roo Code, Goose, OpenHands). Target-specific generators live in `src/eggpool/integrations/`; shared utilities in `src/eggpool/config_utils.py` (`resolve_server_api_key`, `ServerKeyResolution`). Transcoder enablement is persisted to TOML when required. Secret output is controlled explicitly via `contains_secret`. The OpenCode generator emits a `"thinking": "supported"` annotation on models with confirmed thinking capability, omitting the field for `unknown`/`unsupported`/`mixed`/`conflicting` statuses. See `architecture/README.md` ¬ß Package Structure.

Phase 4 adds JSON API endpoints (`GET /api/model-info`, `GET /api/model-info/{model_id}`, `GET /api/model-info/{model_id}/aliases`, `GET /api/model-info/sources`, `POST /api/model-info/refresh`), compact `/v1/models` enrichment under `eggpool.model_info`, and dashboard model-info status pills with tooltips. Phase 5 adds Artificial Analysis and Hugging Face source adapters, manual overrides (`[model_info.overrides]`), alias expansion (`[model_info.aliases]`), source health hardening (`rate_limited_until`, `last_status_code`, `last_payload_count`, `last_success_duration_ms`), richer summary generation (benchmarks, HF metadata, conflicts), and `model_info_overrides` table (migration `0037`). The corrective pass (Phases A‚ÄìF) adds configured-alias seeding at startup (`seed_configured_aliases()` in `src/eggpool/model_info/service.py`), observation-driven canonical detail (`build_canonical_detail()` with a nested `detail["limits"]` block that holds `effective_context`, `external_context`, `effective_output`, `external_output`, plus a legacy flat-key back-compat reader), single-model refresh (`refresh_model_info(model_id, *, source=None, force=False)` exposed via `POST /api/model-info/refresh?model_id=...&source=...&force=...`), catalog-complete Models page rows that merge `_get_catalog_rows()` against request stats before rendering, and a one-shot `backfill_legacy_detail_blocks()` startup step that lifts pre-Phase-B flat-key detail rows into the normalized nested schema. See `architecture/README.md` ¬ß Model Information.

Protocol compatibility alone does not imply thinking support. Global overrides at `[model_capabilities."<model_id>".thinking]` and provider-scoped overrides at `[providers.<id>.model_capabilities."<model_id>".thinking]`. Provider-scoped overrides take precedence over global. See `architecture/README.md` ¬ß Model Capabilities and `docs/thinking.md` for the operator guide.

Only explicit API-control documentation (e.g. OpenRouter `supported_parameters` listing "reasoning"/"thinking") produces `status = "supported"`; vague "reasoning model" descriptions stay `unknown`. ## Gotchas

- Configuration changes require a service restart; live reload is intentionally not supported
- No pre-commit hooks are configured in this repo; CI runs ruff, pyright, and pytest via GitHub Actions
- **`static_models` is the source of truth for provider-specific protocol** ‚Äî `FAMILY_PROTOCOLS` (`src/eggpool/catalog/protocols.py`) is a global fallback. Providers like `minimax-cn` that serve MiniMax models on the OpenAI-compatible surface **must** ship `[[providers.<id>.static_models]]` rows with `protocol = "openai"`, otherwise the live `/v1/models` fetch resolves `MiniMax-M*` via the `minimax-` family prefix to `anthropic` and the protocol check clears it to `None`, producing `ModelUnavailableError` instead of `ProtocolMismatchError`. Static seeds survive via `ModelCatalogCache._preserve_static_fields` (`src/eggpool/catalog/cache.py:146-187`).
- **Upstream-authoritative suppression**: local quota estimates are advisory by default (`local_quota_mode = "score_only"`). Only upstream-observed failures (429/402/5xx/auth) and explicit operator disablement suppress routing. Switch to `hard_cap` only as an opt-in escape hatch.
- **Catalog non-destructive refresh**: `ModelCatalogCache.update_from_account(authoritative=False, allow_withdrawals=False)` (the default) preserves prior account/model support on every cycle ‚Äî failed, empty, and partial refreshes never de-pool a healthy account. The destructive `mark_account_models_unavailable` step is only reached when the service explicitly sets both flags, gated by `ModelsConfig.catalog_withdrawal_policy`. Under the default `preserve_until_health` policy the flags are never flipped, so `HealthManager` is the only de-pooling mechanism. Static seeds (`_seed_static_models`) inherit the non-destructive default and survive future refresh cycles via `_preserve_static_fields`.
- **Sibling-wins protocol guard**: `_provider_models` is keyed by `(model_id, provider_id)` and **shared** by every account that lists that provider (e.g. all `opencode-go-0001`/`-0002`/`-0003` accounts share one row per model on the `opencode-go` provider). `_preserve_resolved_protocol()` (`src/eggpool/catalog/cache.py`) is invoked on every non-destructive `update_from_account` call: when a per-provider row already has a resolved protocol and the new entry arrives with `protocol=None`, the prior protocol is preserved. This prevents a single sibling's partial refresh from silently de-pooling every account on that provider. The destructive path (`authoritative=True AND allow_withdrawals=True`) intentionally skips the guard.
- **Provider-scoped Models page merge keys**: `_merge_models_with_catalog()` (`src/eggpool/dashboard/routes.py`) dedupes by `(model_id, provider_id)` when `models.collapse_models=false` so an active sibling provider never suppresses an unused sibling's catalog row, and by `model_id` only when `collapse_models=true` so the collapsed view stays one row per base model. Legacy stats rows that omit `provider_id` fall back to `catalog_by_id[model_id]` for diagnostic field lifting but do **not** suppress provider-scoped catalog rows. The `_model_row_key()` helper is the single source of truth for the key computation.
- **Manual refresh accepts provider-suffixed IDs**: `POST /api/model-info/refresh?model_id=<id>` URL-decodes `model_id`, then calls `parse_model_provider()` (the same suffix parser used by `handle_model_detail()`) so `?model_id=gpt-4o/openai` refreshes the canonical `gpt-4o` row and forwards `provider_id="openai"` to `ModelInfoService.refresh_model_info()` for narrower per-provider catalog matching. Unknown suffix fragments fall through as the literal id. The `?source=` filter is validated against an allow-list (`provider_catalog`, `openrouter`, `artificial_analysis`, `huggingface`, `all`/empty ‚Üí all sources); unknown values return HTTP 400 before the service is touched. The response includes both the requested and canonical model IDs (`requested_model_id`, `model_id`) plus the resolved `provider_id`.
- **Model-info thinking capability enrichment**: `SourceModelRecord.thinking_capability` carries optional thinking/reasoning API-control metadata from sources. `build_canonical_detail()` merges these into the canonical detail's `capabilities.thinking` block. Provider catalog data always outranks external source data. When two external sources disagree (e.g. one says "supported", another says "unsupported"), the merged status is set to `"conflicting"` with details preserved in `notes`. `_propagate_enriched_capabilities()` writes the enriched capability back to the catalog cache, skipping entries where the provider catalog is the original source. Only explicit API-control evidence (like OpenRouter `supported_parameters` listing "reasoning" or "thinking") can set `status = "supported"` ‚Äî vague marketing descriptions like "reasoning model" remain `unknown`.
- **Thinking observability counters**: `ThinkingMetricsCounter` (`src/eggpool/metrics/thinking.py`) uses pipe-delimited label keys (e.g. `"transcoded|openai|anthropic|anthropic-prod"`) for low-cardinality tracking. The singleton `get_counter()` is lazily created. Counters are in-memory only and reset on restart ‚Äî they complement the durable `usage_rollups` table which tracks token-level metrics. The `thinking_trace_json` column on `requests` stores the per-request structured trace for diagnostic inspection via `/api/stats/recent/{id}`.
- **Capability-aware routing**: `classify_thinking_request()` in `src/eggpool/catalog/capabilities.py` inspects the request body for OpenAI `reasoning_effort` / `reasoning` and Anthropic `thinking` / `thinking_budget` indicators plus assistant history `reasoning_content` blocks. The `ThinkingRequestRequirement` result is threaded through `get_eligible_accounts()` ‚Üí `Router._selection_candidates()` ‚Üí `select_account()` / `select_accounts_for_failover()`. Each candidate's thinking capability status is checked against `[transcoder.capability_policy]` settings (`unsupported_thinking`, `unknown_thinking`, `mixed_collapsed_thinking`). Default policy is `reject` for all ‚Äî a client explicitly asking for thinking gets either a compatible upstream or a clear `CapabilityError` (HTTP 400). `mixed_collapsed_thinking = "filter"` silently drops non-thinking providers in collapsed multi-provider models; if no supported providers remain, the original list is returned (falling through to rejection). `conflicting` status is always rejected ‚Äî operators resolve conflicts via manual overrides. Requests without thinking controls route exactly as before. `CapabilityError` is distinct from `ModelNotFoundError` (404) and `ModelUnavailableError` (503).
- **Models page link URL encoding**: `render_models()` percent-encodes the detail-link path segment via `urllib.parse.quote(safe="")` so model ids containing provider suffixes (`/`) or query/HTML metacharacters (`?`, `#`, `<`, `>`, `"`) survive the round-trip through `/models/{model_id:path}` + the detail handler's `unquote()`. The stats-row merge preserves the literal model id; only the emitted `<a href>` is percent-encoded.
- **Info-status filter aliases**: the dashboard's `?info_status=` filter accepts both canonical status names (`sparse_new`, `conflicting`, `source_unavailable`, `manual_override`) and the display aliases the compact `/api/model-info` summary exposes (`sparse`, `conflict`, `source-unavailable`, `manual`). `_normalize_info_status_filter()` in `src/eggpool/dashboard/routes.py` owns the mapping; new statuses only need to be added there and in `_STATUS_ALIASES`.
- **Same-tier fairness rotor**: EggPool is not purely lowest-score-wins for same-tier peer accounts. When accounts are effectively tied by priority, weight, health, protocol, and utilization score, same-tier fairness rotates candidates to avoid stable config-order bias and subscription starvation. When `fairness_mode = "round_robin"` (the default), effectively tied accounts within the same priority tier are rotated deterministically via `FairnessRotor` (`src/eggpool/routing/fairness.py`). Band membership requires same priority, same weight, same transcode status, and score within `fairness_epsilon` of the best. The server runtime honors all `[routing]` fairness config fields: `fairness_mode`, `fairness_epsilon`, and `fairness_scope`. The `fairness_scope` controls rotation group granularity: `provider_model_protocol` (default) includes the routed protocol in the key, `provider_model` intentionally collapses protocol groups, and `priority_model_protocol` intentionally co-balances same-priority providers serving the same model. The rotor's position map is capped at 4096 entries (`_ROTOR_HARD_CAP`); when reached the entire map is cleared and rotation restarts from 0. Restart also resets all rotor state ‚Äî durable round-robin is explicitly out of scope.
- **Backoff persistence**: upstream-derived backoffs survive restarts via the `account_backoffs` table (`src/eggpool/db/schema/0024_account_backoffs.sql`). Local cost overruns must never be persisted as backoff rows.
- **Synthetic 503 vs 502**: `ModelUnavailableError` (503) is reserved for genuine pre-dispatch unavailability. `UpstreamExhaustedError` (502) is raised when every candidate account was attempted and exhausted mid-request.
- **Streaming finalizer shielding**: streaming `_build_stream_generator` finalization runs under `asyncio.shield(asyncio.wait_for(..., timeout=10))` so ASGI task cancellation cannot kill the finalizer while it holds the DB lock. Leaks that escape this path are caught by the periodic `stale_request_finalizer` background task (`app._finalize_stale_requests`, runs every 60s).
- **model_info FK seeding**: `model_info_canonical.model_id`, `model_info_observations.model_id`, `model_info_aliases.model_id`, and `model_info_overrides.model_id` all carry FKs to `models.model_id` (`src/eggpool/db/schema/0036_model_info.sql`, `0038_model_info_phase5.sql`). Catalog entries for unresolved static models can reach the model-info paths before `_persist_catalog` writes them to the `models` table (and pre-0036 installations may have FK-violating rows for any reason). The repository seeds a placeholder `models` row inside the same transaction as the sidecar write ‚Äî see `upsert_canonical_with_model` and `upsert_observation` in `src/eggpool/model_info/repository.py`. `INSERT OR IGNORE` is mandatory: catalog-authored fields (display_name, protocol, capabilities) must never be clobbered by an observation-side seed.
- **Routing-decision score components** (`RoutingDecisionTrace.score_components`) carry the per-account score breakdown on every persisted `routing_decisions` row (`score_components_json`, migration `0035`). The dashboard and `eggpool accounts explain` consume this directly; do not re-score from quota tables when only the diagnostic breakdown is needed.
- **`_select_lock` publish ordering**: in `RequestCoordinator._select_and_persist_attempt()`, runtime publication (`Router.increment_active_request_count` + `QuotaEstimator.add_reservation`) must run INSIDE `_select_lock` AFTER the durable transaction commits but BEFORE the lock releases. The two contexts are explicit nested `async with` blocks (outer `_select_lock`, inner `_db.transaction()`). Key invariant: block placement (publication outside DB transaction body, inside `_select_lock`), not context-exit order.
- **`db.transaction()` nesting semantics** (`src/eggpool/db/connection.py`): nesting is detected via SQLite's per-connection `conn.in_transaction`, NOT task identity. A shielded or `create_task` child entering `db.transaction()` while the parent's `BEGIN IMMEDIATE` is still open piggybacks on the outer's commit boundary; the child never re-issues `BEGIN` and never acquires `_connection_lock` (the outer holds it). This fixes the AB/BA deadlock where a shielded child would block waiting for `_connection_lock` while the parent awaited `asyncio.shield()`. Two ContextVars back the semantics: `_in_transaction_context: ContextVar[bool]` is set/reset by both nested and outermost paths and is inherited by shielded/child tasks so they can `execute_write`; `_transaction_owner: ContextVar[Task]` is set only by the outermost path and is used by `vacuum()` to refuse running when the *current* task holds the lock (deadlock guard). `vacuum()` and `_require_transaction_owner()` consult `_in_transaction_context`, not `_transaction_owner`, so shielded/child task writes piggyback cleanly. Regression coverage lives in `tests/unit/test_database.py::TestTransactionNestingAcrossTaskBoundaries`.
- **`eggpool accounts explain` reads from SQLite, not an empty cache**: the command hydrates the catalog via `ModelCatalogCache.hydrate_from_db(db)` from `models`/`provider_model_metadata`/`account_models` rows. A thin `_CatalogShim` exposes the loaded cache as a `CatalogService`-compatible object. Output uses `click.echo` (no `rich` dependency).
- **Startup crash recovery**: `_crash_recovery` runs at every startup and recovers ALL pending requests and active reservations with no time threshold. A process restart is a definitive boundary.
- **Pricing pipeline**: prices flow TOML override ‚Üí upstream metadata ‚Üí external catalog (OpenRouter / OpenCode Zen via the alias registry). Cost precedence: `provider_reported > derived/partial/exact > estimated > unknown`.
- **`eggpool stats recompute-costs [--dry-run|--apply] [--limit N]`**: recomputes cost from current price snapshots. Default `--dry-run`. Implemented in `src/eggpool/cost_recompute.py`.
- **Automatic backups**: in-process daily backups run by default under the `automatic_backup` supervised task (`src/eggpool/background/backup.py`). Controlled by `[backup]` config section.
- **DNS cache**: `OutboundClientManager` and `ProviderClientPool` both integrate a `DnsNetworkBackend` that caches resolved DNS entries. Controlled by `[network.dns_cache]` (enabled by default, TTL 1800s, max 50 entries). Exposes precise counters and derived rates for operator diagnostics.
- **Memory footprint caps**: every growth axis is bounded by hardcoded caps (`EWMA_HARD_CAP = 4096`, `GLOBAL_EWMA_HARD_CAP = 1024`, `MAX_TRACKED_HOSTS = 256`). Regression gate: `tests/integration/test_memory.py` (`pytest.mark.slow`). See `plans/memory.md` for the full design.
- **Transcoder body translation**: `select_transcoder()` in `src/eggpool/transcoder/protocol.py` is the single source of truth for translator dispatch. Loss-of-information warnings are accumulated on `TranscodeContext.loss_warnings` and logged at request completion.
- **Transcoder policy wiring**: `RequestCoordinator` receives `config.transcoder` (a `TranscoderPolicy`) via the `transcoder_policy` constructor parameter at app startup (`src/eggpool/app.py`). The coordinator stores it as `self._transcoder_policy` and reads `.features` to gate per-feature transcoding (thinking, tools, vision, etc.) during body encoding/decoding and `.enabled` to check the legacy escape hatch. `app.state.transcoder_policy` also remains set for preflight helpers and diagnostics in `proxy_request.py`. When constructing a `RequestCoordinator` in tests, pass an explicit `transcoder_policy` or assert the desired default; never rely on implicit `None`.
- **Thinking budget resolver**: `resolve_thinking_budget()` in `src/eggpool/transcoder/budget_resolver.py` is the single source of truth for effort-to-budget translation. Resolution order: explicit `thinking.budget_tokens` (Anthropic) ‚Üí `reasoning_effort` (OpenAI) via `ThinkingCapability.effort_to_budget_tokens` ‚Üí `[transcoder.thinking_budget_defaults]` ‚Üí hard-coded fallback. The coordinator looks up the thinking capability from the catalog cache by `model_id` at encode time; provider-specific overrides are best-effort since the upstream provider is not yet resolved. `budget_resolution_policy = "strict"` rejects unknown efforts and clamped budgets with `BudgetResolutionError` (HTTP 400).
- **Selected-provider thinking budget override (Phase H ‚Äî final cleanup)**: `RequestCoordinator._apply_selected_provider_transcode_adjustments()` is the single entry point that re-runs `resolve_thinking_budget()` against the selected provider's `ThinkingCapability` after route selection. The recompute parses `context.original_body` via `_extract_original_thinking_budget_inputs()` so the resolver sees the **original** client intent (OpenAI `reasoning_effort` or Anthropic `thinking.budget_tokens`), not the already-translated upstream budget. Forwarding the translated `thinking.budget_tokens` as `requested_budget_tokens` would short-circuit the effort mapping because the resolver prioritises `requested_budget_tokens` over `requested_effort`. Streaming and non-streaming dispatch paths use the same helper, so provider-scoped `effort_to_budget_tokens` overrides win at dispatch for collapsed model ids.
- **Post-selection capability rejection cleanup (Phase H)**: `_finalize_selected_capability_rejection()` is the cleanup contract for any post-selection `CapabilityError` (e.g. provider-specific strict-clamp rejection). It finalizes the attempt row with `release_reason = "capability_rejected"` and `retry_category = "never"`, releases the reservation (durable + in-memory), decrements `Router.active_request_count`, releases the `HealthManager` probe slot, stamps `thinking_trace.decision = "rejected"` with the rejection reason, finalizes the request row as `client_error` with `thinking_trace_json`, and bumps the rejected thinking counter. **No upstream health penalty is recorded** ‚Äî a capability rejection is a client-validation failure, not an account health signal. Streaming and non-streaming dispatch paths both wrap the helper in a `try/except CapabilityError` around `_apply_selected_provider_transcode_adjustments()` so neither path leaks attempt state, reservation cost, active count, or health slots.
- **Closing-pass thinking hardening (Phases A‚ÄìG)**: `extract_thinking_status_from_entry()` in `src/eggpool/catalog/capabilities.py` treats missing `capabilities.thinking` metadata as `unknown` (Phase A), so the configured `[transcoder.capability_policy].unknown_thinking` policy applies uniformly across routing decisions. `BudgetResolutionError` is a subclass of `CapabilityError` (Phase B), so the proxy layer's existing handler renders it as HTTP 400. After route selection, `RequestCoordinator._recompute_thinking_budget_for_selected_provider()` re-runs the budget resolver against the **selected provider's** capability so provider-scoped overrides take effect at dispatch (Phase C). `classify_thinking_warning_decision()` collapses the loss-warning list into a single `decision` field on `thinking_trace` (`passthrough` / `transcoded` / `dropped` / `clamped` / `rejected`); `_determine_thinking_rejection_status()` attributes routing rejections to `unknown` vs `unsupported` (Phase D). `classify_thinking_request()` also detects top-level `reasoning_content` on assistant messages (Phase E). The vestigial `supports_tools: True` is removed from `model_capabilities_to_dict()` (Phase F). Anthropic top-level `thinking` blocks dropped during Anthropic‚ÜíOpenAI transcoding emit an explicit `anthropic_top_level_thinking_dropped` warning kind instead of the generic `dropped_field` bucket (Phase G). See `docs/thinking.md` ¬ß11 for the operator guide.
- **Agent config generation**: `eggpool configsetup` dispatches to target-specific generators in `src/eggpool/integrations/`. Shared utilities (model ID formatting, base URL defaults, clipboard/write logic) live in `src/eggpool/config_utils.py`. When adding a new target, create a new module in `integrations/` and register it in the Click command; reuse the shared utilities rather than reimplementing them.

## Error Handling

Use the hierarchy in `errors.py`. Chain exceptions with `raise ... from err` or `raise ... from None`.

- `AggregatorError` ‚Üí `ConfigError`, `DatabaseError`, `ProxyError`
- `UpstreamError` (has `status_code`) ‚Üí `TemporaryUpstreamError`, `TransientUpstreamError`, `AuthenticationError`, `QuotaExhaustedError`, `RateLimitError` (has `retry_after`), `ModelUnavailableError`
- `ModelNotFoundError` (has `model_id`), `NoEligibleAccountError`, `CatalogUnavailableError`, `AuthenticationUnavailableError`, `UpstreamExhaustedError`, `AccountSuspendedError`, `RequestTooLargeError`, `ModelInfoSourceFetchError`, `ContextLimitExceededError`, `CapabilityError`

## Fast-Path CLI

- `src/eggpool/cli.py` is a tiny bootstrap (74 lines)
- `main()` calls `eggpool.fastcli.maybe_run_fast_command()` first; recognized fast commands (`croncheck`, `ensure-running`) are dispatched without importing Click
- **Do not add transitive imports to `runtime_paths` or `fastcli`** ‚Äî they are stdlib-only and must stay lightweight for the Raspberry Pi watchdog contract
- Unrecognized commands fall through to `eggpool.cli_full`, which holds the heavy Click CLI
- Public symbols (`cli`, helpers used by tests) are lazily forwarded from `cli_full` via PEP 562 `__getattr__` ‚Äî so `from eggpool.cli import cli` and existing test imports still work without loading the full graph
